# Exercise 4 – Korrektur der Exception-Behandlung

## Was wurde geändert?

In diesem Branch wurde die Exception-Behandlung rund um die Datenbank-Initialisierung korrigiert.

Betroffene Dateien:

- `DatabaseUtil.java`
- `Main.java`

---

## Ausgangsproblem

In Exercise 3 hatten wir teilweise noch `SQLException` in höheren Schichten wie `Main` oder `MovieService` verwendet.

Das war nicht sauber, weil `SQLException` ein technisches Detail der Datenbankschicht ist.

Der Fehler war:

```java
catch (SQLException e)
````

an Stellen, wo eigentlich keine `SQLException` mehr direkt geworfen wurde oder wo höhere Schichten nicht direkt mit SQL-Fehlern arbeiten sollten.

Dadurch entstand z. B. die Fehlermeldung:

```text
Exception java.sql.SQLException is never thrown in the body of the corresponding try statement
```

---

## Änderung in `DatabaseUtil.java`

Vorher:

```java
public static void initializeDatabase() throws SQLException
```

Nachher:

```java
public static void initializeDatabase() throws DatabaseException
```

Die `SQLException` wird jetzt direkt in `DatabaseUtil` abgefangen und in unsere eigene `DatabaseException` umgewandelt:

```java
catch (SQLException e) {
    throw new DatabaseException("Could not initialize database", e);
}
```

Dadurch bleibt SQL-Fehlerbehandlung in der Datenbankschicht.

---

## Änderung in `Main.java`

Vorher wurde in `Main` direkt `SQLException` behandelt:

```java
catch (SQLException e)
```

Jetzt behandelt `Main` nur noch unsere eigene Exception:

```java
catch (DatabaseException e)
```

Dadurch kennt `Main` keine SQL-spezifischen Details mehr.

---

## Warum ist das besser?

Die Fehlerbehandlung ist jetzt sauberer getrennt:

```text
SQLException
→ wird in DatabaseUtil abgefangen
→ wird zu DatabaseException
→ wird in Main behandelt
```

Das verbessert die Struktur des Codes, weil höhere Schichten nicht direkt von SQL abhängig sind.

---

## Bezug zu Exercise 4

Diese Änderung passt gut zu Exercise 4, weil sie die Architektur verbessert.

Besonders relevant für:

* **Separation of Concerns**
  Datenbankfehler werden in der Datenbankschicht behandelt.

* **Single Responsibility Principle**
  `DatabaseUtil` ist für Datenbankinitialisierung zuständig.
  `Main` ist nur für den Programmstart zuständig.

* **Dependency Inversion / geringeres Coupling**
  Höhere Schichten hängen nicht mehr direkt von `SQLException` ab.
